# Manila — 马尼拉桌游模拟器

一款基于 [马尼拉桌游](rules.md) 规则实现的本地 GUI 模拟器，支持 3–5 人混合对战（人类玩家 + AI 对手）。

## 功能特性

- **图形界面**：基于 pygame，分辨率 1280×800，支持货船轨道、棋盘、市场、玩家状态实时显示
- **混合对战**：1–5 名真人玩家 + AI 对手，自由配置
- **完整事件日志**：所有关键行动（竞价、部署、掷骰、结算）自动记录并以彩色呈现
- **存档/读档**：随时保存当前游戏，下次继续；读档后 AI 行为完全复现（基于随机种子）
- **部署回滚**：派遣工人后可一键撤销本小轮所有部署重新决策
- **主菜单** + 全屏日志查看器

## 快速开始

```bash
cd Manila
python3 gui_main.py
```

启动后进入主菜单，选择**开始游戏**即可。

### 打包为可执行文件

需要先安装 `pyinstaller`（脚本会自动安装）：

```bash
pip install pyinstaller pygame
```

**Linux**

```bash
bash build_linux.sh
# 生成 bin/linux/manila
```

**Windows**（在项目目录中双击或在 cmd 中运行）

```bat
build_windows.bat
:: 生成 bin\windows\manila.exe
```

也可以直接使用 spec 文件构建（跨平台）：

```bash
pyinstaller gui_main.spec
# 输出至 dist/manila（Linux）或 dist/manila.exe（Windows）
```

> **注意**：首次构建时 PyInstaller 会分析所有依赖，耗时约 1–3 分钟。
> `dist/` 和 `build/` 目录可以安全删除，可执行文件在 `bin/linux/` 或 `bin\windows\` 下。

## 操作指南

### 主菜单

| 按钮 | 描述 |
|------|------|
| 开始游戏 | 进入玩家配置页，开始新游戏 |
| 继续游戏 | 打开存档文件，继续上次游戏 |
| 帮助 | 查看简要规则说明 |
| 退出 | 关闭程序 |

### 游戏配置页

1. 选择**玩家人数**（3–5 人）。
2. 为每位玩家输入名称，并选择**人类**或 **AI**。
3. 点击**开始游戏**。

### 游戏界面布局

```
+---------------------------------+----------------+
|           货船轨道               |   市场股价      |
+---------------------------------+                |
|           棋盘位置               +----------------+
|  港口 造船厂 航海家 海盗 保险    |   玩家状态      |
|                                 +----------------+
|                                 |   事件日志      |
+---------------------------------+----------------+
|              行动面板（玩家操作区）                 |
+---------------------------------------------------+
```

### 行动面板操作

- **竞价**：用 `-10/-5/-1/+1/+5/+10` 按钮调整金额，点击**出价¥X** 确认；点击**放弃**跳过。
- **选择货物**：点击要**排除**的货物按钮（该货物本轮不运输）。
- **设定起始位置**：为每艘货船用 +/- 调整格数，总和须等于 9。
- **派遣工人**：点击目标位置（货船槽/港口/造船厂/航海家/海盗/保险）；点击**放弃**跳过本大轮剩余派遣。
- **航海家移动**：分步点击目标货船及方向，完成后点击**确定**。
- **海盗**：选择登船目标 → 选择踢出位置（若船满）→ 选择送往港口或造船厂。

### 顶部按钮栏（游戏中）

| 按钮 | 功能 |
|------|------|
| 保存 | 将当前游戏保存为 `.json` 存档 |
| 加载 | 打开一个存档文件（当前游戏将终止） |
| 日志 | 打开全屏日志查看器（鼠标滚轮滚动，ESC 关闭） |

## 存档与读档

- 存档文件保存在 `saves/` 目录，格式为 JSON。
- 存档内容包含：玩家配置、随机种子、所有历史决策记录。
- 读档时程序会用相同的随机种子重放所有 AI 决策，保证游戏状态完全复现。
- **注意**：旧版存档（无 `seed` 字段）读档后 AI 行为可能与存档时略有差异。

## AI 说明

AI 采用**贪心策略**，具体行为如下：

| 阶段 | AI 策略 |
|------|---------|
| 竞价港务长 | 35% 概率直接放弃；否则出比当前最高价多 1–5 的随机值（不超过自身资金） |
| 购买股票 | 优先购买"持仓评分"最高的货物股票；30% 概率跳过 |
| 选择货物 | 排除持仓评分最低的货物 |
| 设定起始位置 | 将自持股最多的货物放在最优位置，其余均分 |
| 派遣工人 | 优先投入高评分货船；资金不足时质押最差股票贷款 |
| 航海家 | 将最有价值的货船尽量推向港口 |
| 海盗 | 优先抢劫货值最高的货船，送往港口或造船厂取决于持仓情况 |

## 配置说明

所有规则参数均在 `data/config.json` 中，无需修改源码即可调整：

```jsonc
{
  "game": {
    "initial_money": 30,          // 初始资金
    "ship_start_sum": 9,          // 货船起始位置总和
    "ship_track_length": 13,      // 轨道总长（0–13）
    "price_step": 5,              // 每次股价上涨幅度
    "end_price": 30,              // 触发游戏结束的股价
    "min_stock_price": 5,         // 股票最低购买价
    "loan_amount": 12,            // 质押贷款金额
    "loan_redeem": 15             // 赎回质押股票费用
  }
}
```
